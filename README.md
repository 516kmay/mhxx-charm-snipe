# MHXX Charm Snipe Tool

モンスターハンターダブルクロス（MHXX）の **お守りスナイプ（RNG操作）** を支援する Java Swing 製デスクトップツール。

お守り検索・乱数逆算・調合スナイプ・Arduino自動化コード生成・タイマー機能を1つのアプリに統合している。

---

## 動作環境

- Java 21 以上
- Switch版MHXX（実機RNG挙動に基づく）
- Arduino機能を使う場合: Arduino Leonardo + NintendoSwitchControlLibrary

## 起動方法

### Windows
```
run.bat
```
内部で `javac` ビルド → `java` 実行 を行う。初回はビルドのため少し時間がかかる。

### macOS / Linux
```
run.sh
```

## 全体ワークフロー

このツールは「**狙ったスキル・SP値・スロットを持つお守りを、特定のRNGフレームに到達することで意図的に出す**」という目的のために作られている。

典型的な利用フローは以下：

1. **お守り検索** で目的のお守りが出現するフレームを見つける
2. **周辺表示** で目標フレーム周辺の安全マージン（ロードブレ耐性）を確認
3. **キャリブレーション** で実機の消費フレーム値（c, fc）を実測
4. **調合スナイプ** または **Arduino** タブでスナイプ用のArduinoコードを生成
5. 実機で実行 → 録画確認
6. ズレがあれば **補正計算** で次回パラメータを再計算

## タブ別機能解説

### 1. お守り検索

目標のお守り条件（第1スキル・SP・第2スキル・SP・スロット数）を指定して、それを満たすフレームを総当たりで探索する。

**主な入力:**
- お守り種別（風化／古びた／光るお守り）
- 原産地（マカ錬金／クエスト炭鉱）
- カテゴリフィルタ（攻撃系・防御系・剣士用など）
- 第1スキル＋SP値、第2スキル＋SP値、スロット数
- 検索範囲（フレーム数、デフォルト 1,000,000F）
- 検索モード（完全一致／以上）

**出力:**
- 該当フレームのリスト（フレーム番号・スキル・SP・スロット・累計時間・レア度）
- 各行を右クリックで「Arduinoコード生成」が可能

### 2. 周辺表示

特定のフレーム位置を中心に、その前後 N フレームのお守り内容を一覧表示する。

「狙ったフレームでハズレを引いた時、隣のフレームが何だったか」を確認するのに使う。

**特殊なハイライト:**
- 中央フレーム → 強調色
- ±30F以内 → 「ロードブレ耐性圏内」として薄くハイライト

### 3. 調合スナイプ

「調合（アイテム生成）」を高速で連打することで RNG を消費し、目標フレームに到達する手法に対応した計算と Arduino コード生成。

**入力:**
- 累積弾数（実機の調合録画から逆算）
- 目標フレーム（お守り検索や有名お守りタブから連動）
- 調合回数の目安、Lv2弾までの↓キー数

**出力:**
- 調合スナイプ Arduino コード（録画用）
- 周辺フレームのお守り一覧（誤差耐性確認用）

**目標Fハイライト機能:**

逆算結果のテーブルで、目標F以下で最も近い候補（推奨フレーム）を**緑色背景**で強調表示する。目標Fを超過する候補は赤系の文字色で警告。これにより複数候補から最適なフレームが一目で分かる。

### 4. 調合Arduino（2コード方式）

調合スナイプを **2段階のArduinoコード** で実装する。HOME中断中は乱数が停止する仕様を利用。

**ワークフロー:**
1. **コード1** → Continue連打 → 自宅移動 → 調合（10秒A長押し） → 30秒録画 → HOME中断
2. 手動で録画確認 → 累積弾数から現在フレームを特定
3. **コード2** → L連打でコントローラ起こし → HOME復帰 → マカ錬金準備 → 投入確認画面で待機 → 投入確定 → ケルビ納品 → 鑑定

**コントローラ起こしについて:**
HOME中断→復帰時、Switch側のコントローラ再認識が完了する前の最初の入力が無視されることがある。これを回避するため、コード2の冒頭で**Lボタンを5連打**してから操作を開始する設計。

**入力フィールド:**
- Continue回数（コード1）
- 調合回数の目安
- Lv2弾までの↓数
- 現在F（消費後）
- 目標F（調合スナイプタブと双方向連動）

### 5. タイマー

#### ラップタイマー
- スペースキーで開始／ラップ記録
- Enterキーで停止＆記録
- Rキーでリセット
- ラップタイム・累計時間・フレーム換算を表示
- CSV保存対応

#### 目標時間設定
- 目標フレーム・基準フレームから所要時間を計算

#### マカ錬金カウントダウン（手動投入用）
Arduinoを使わず手動でマカ錬金のタイミングを取りたい時のカウントダウン機能。

- **待機時間 (ms)**: 待ちたい時間（調合スナイプの wait_ms をコピペ可）
- **開始までの猶予 (秒)**: 準備時間
- **切り上げ (秒)**: 投入を少し早める時間
- **サウンド有効**: チェック時、残り10秒からビープ、終了時に強い音

### 6. Arduino（Continue連打法）

タイトル画面でContinue→キャンセルを連打して RNG を進めるシンプルな手法。調合スナイプより手軽だが、目標Fが大きい場合は時間がかかる。

**入力:**
- 目標フレーム
- 基本消費フレーム c（デフォルト 1,675）
- Continue1回の消費フレーム fc（デフォルト 714）
- 待機時間の範囲（5,000〜20,000ms など）

**出力:**
- 計算結果: Continue連打回数、端数待機ms、推定到達F、総所要時間
- Arduino スケッチ（コピー or `.ino` 保存）

#### 待機時間の補正
実測フレームと目標フレームのズレから、次回用の Continue回数 と 待機時間 を再計算する。

実機の実測値から fc_actual を逆算し、前回 nc 近傍で探索して、最も差分が小さい解を採用する設計。

**入力:**
- 目標フレーム、実測フレーム、Continue回数、前回の待機時間

**出力:**
- ズレ量（オーバー／不足）
- 補正後パラメータ（Continue回数、待機時間）
- 補正ログテーブル（CSV保存可）

### 7. 報酬逆算

クエスト報酬画面に表示されたアイテムの個数と並びから、現在のフレームを特定する。

調合スナイプでズレた時、「どのフレームで終わったのか」を逆算するのに使う。

**入力:**
- 通常報酬枠数
- 報酬アイテムリスト（複数可）

**出力:**
- 候補フレームと報酬パターンの一致度

### 8. クエスト周回

特定のクエスト報酬パターンを利用したRNG消費＋逆算ワークフロー用。

### 9. キャリブレーション

実機の消費フレーム値（c, fc）を実測するための補助タブ。

### 10. 有名お守り

実用性の高い「有名な良お守り」のリストとそのフレーム位置を確認できる。

## 設定の保存

ウィンドウを閉じる時に `~/MHXXCharmAppSettings.properties`（ホームディレクトリ）に各種入力値が自動保存される。次回起動時に復元される。

**保存対象:**
- お守り検索: 種別・スロット・検索範囲・選択スキル・SP値
- 周辺表示: フレーム位置・原産地・前後範囲
- 報酬逆算: 追加報酬閾値
- 調合スナイプ: 累積弾数・目標F・Continue回数・調合回数・Lv2↓数
- 調合Arduino: 現在F・目標F（調合スナイプの目標Fと連動）
- 待機時間の補正: 目標F・実測F・Continue回数・前回の待機時間
- Arduino: c・fc・目標F
- ウィンドウ位置・サイズ
- 選択中タブ

## 補正計算ロジック（技術的補足）

ツールが目標到達するフレームを計算する基本式：

```
actual_frame = c + fc × nc + 0.030 × t2
```

- `c`: 基本消費フレーム（固定）
- `fc`: Continue1回あたりの消費フレーム
- `nc`: Continue連打回数
- `t2`: 端数待機時間（ms）
- `0.030`: 1msあたり0.03フレーム（30fps）

実機との誤差が出た場合、`fc_actual` を逆算してから再計算する：

```
fc_actual = (actual - c - 0.030 × prevT2) / prevNc
```

これを使って前回 `nc` 近傍から探索し、最小変更で目標到達できる解を採用する。
